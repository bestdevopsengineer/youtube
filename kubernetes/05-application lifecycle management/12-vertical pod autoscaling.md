# 1- VPA admission controller
    intervenes the pod creation process and uses the
    recommandations from the recommander again
    to mutate the pod spec to apply the recommanded 
    CPU and memory values at startup, so the newly created pods 
    start with the correct resource requets
    so it collects info from the updater monitor,
# 2- VPA updater
    detect pods that are running with suboptimal resources
    and evicts them when an update is needed
    get info from the recommander and monitor a pod
    and if a pod needs updated, it evicts them
    
# 3- VPA recommender
    contnuously monitoring resource usage from k8s metric API
    and collect historical and live usage data for pods
    provide recommandations on optimal CPU and memory values
    doesnt modify the pod directly only suggests changes
